# 04. カスタマイズ方法

`translations/ja.json` を編集することで、翻訳を追加・変更・削除できます。
コードの知識がなくても、JSONファイルの基本的な書き方さえ理解すれば誰でも編集できます。

---

## ja.json の基本構造

```json
{
  "英語のテキスト": "日本語のテキスト",
  "Syncs": "同期",
  "Models": "モデル",
  "Running": "実行中"
}
```

- **左側**（コロンの前）: Hightouch UIに表示される英語のテキスト（大文字・小文字・スペースも完全一致）
- **右側**（コロンの後）: 置き換える日本語のテキスト

---

## ファイルの並び順について

`ja.json` のエントリは **A-Z アルファベット順**（英語キー基準）で管理しています。

新しい翻訳を追加する際は、アルファベット順の正しい位置に挿入してください。ファイルの並び順は翻訳の動作には影響しませんが、重複チェックや編集のしやすさのために順序を維持することを推奨します。

---

## 翻訳を追加する

### 方法 A: テキストエディタで直接編集

1. `translations/ja.json` をテキストエディタ（メモ帳・VS Code など）で開く
2. アルファベット順の適切な位置に新しい行を追加する
3. `"英語テキスト": "日本語テキスト",` の形式で記述する
4. ファイルを保存する
5. Chrome/Edge の拡張機能管理ページでリロードする（後述）

### 例：「Overview」を「概要」に翻訳したい場合

```json
{
  "Syncs": "同期",
  "Models": "モデル",
  "Overview": "概要"    ← O の位置に追加
}
```

> ⚠️ **JSON の書き方のルール**
> - 各行の末尾にカンマ `,` が必要です（**最後の行だけカンマ不要**）
> - 文字列はダブルクォート `"` で囲む
> - 全角文字は使わない（コロン `:` は必ず半角）

---

### 方法 B: AI（Claude・Gemini 等）を使って編集する

`ja.json` を AI サービスにアップロードして、自然言語で指示する方法です。エンジニアでなくても翻訳の追加・修正が行えます。

**手順**

1. `translations/ja.json` を Claude・Gemini・NotebookLM 等にアップロードする
2. 以下のような指示を送る

**指示文テンプレート:**

```
このJSONファイルはHightouch UIの英語→日本語翻訳辞書です。
以下のルールに従って翻訳を追加・修正してください：

【ルール】
- キー（左側の英語）は絶対に変更しない。値（右側の日本語）のみ編集する
- 固有名詞（Hightouch / Snowflake / BigQuery / Salesforce 等の製品名）は翻訳しない
- A-Zアルファベット順（英語キー基準）を維持する
- 同じキーが既に存在する場合は追加しない

【依頼内容】
（例）「Pipeline」を「パイプライン」に翻訳するエントリを追加してください。
```

3. AIが修正したJSONを出力したら、ファイルに上書き保存する
4. 拡張機能をリロードして反映を確認する

> **注意**: AIが出力したJSONは必ず内容を確認してから保存してください。特に `_comment` や `_templates` の先頭セクションが削除されていないか確認が必要です。

---

## 翻訳を変更する

既存の翻訳が気に入らない場合は、右側の日本語テキストを書き換えます。

**変更前:**
```json
"Destination": "送信先",
```

**変更後:**
```json
"Destination": "配信先",
```

---

## 翻訳を削除する

該当の行ごと削除します。削除後、前の行の末尾カンマが不要にならないか確認してください。

---

## 変更を拡張機能に反映する方法

ファイルを編集・保存しただけでは拡張機能に反映されません。以下のいずれかの手順が必要です。

### 方法 A: 拡張機能をリロードする（推奨）

1. `chrome://extensions`（または `edge://extensions`）を開く
2. 「Hightouch 日本語化」の更新アイコン（🔄）をクリック
3. `app.hightouch.com` のタブをリロードする（Ctrl+R または ⌘+R）

### 方法 B: ページをリロードする

`app.hightouch.com` のタブをリロードするだけで反映される場合があります（キャッシュにより反映されない場合は方法Aを試してください）。

---

## 長い文字列を優先する理由

辞書に以下のような登録があるとします：

```json
"Sync now": "今すぐ同期",
"Sync": "同期"
```

もし短い `"Sync"` を先に処理すると、「Sync now」の「Sync」だけが「同期」に変換され「同期 now」になってしまいます。

このため、この拡張機能は**長い文字列のキーを先に処理する**設計になっています。`ja.json` に追加する順番を気にする必要はありません。

## カスタマイズを他のユーザーと共有する

良い翻訳を追加できた場合は、ぜひ Pull Request でメインリポジトリに貢献してください。詳細は [06. 貢献・共有方法](./06_contributing.md) を参照してください。

## 自社専用の翻訳辞書を持つ

リポジトリをフォークすることで、自社専用の翻訳辞書を独立して管理できます。例えば「Destination」を業界慣習に合わせて「媒体」と訳したい場合など、フォーク版でカスタマイズすることで対応できます。

フォーク方法は [06. 貢献・共有方法](./06_contributing.md) を参照してください。
