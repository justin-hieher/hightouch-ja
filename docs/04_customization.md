# 04. カスタマイズ方法

`translations/ja.json` を編集することで、翻訳を追加・変更・削除できます。
コードの知識がなくても、JSONファイルの基本的な書き方さえ理解すれば誰でも編集できます。

---

## ja.json の基本構造

```json
{
  "英語のテキスト": "日本語のテキスト",
  "Syncs": "同期",
  "Models": "モデル",
  "Running": "実行中"
}
```

- **左側**（コロンの前）: Hightouch UIに表示される英語のテキスト（大文字・小文字・スペースも完全一致）
- **右側**（コロンの後）: 置き換える日本語のテキスト

---

## 翻訳を追加する

### 手順

1. `translations/ja.json` をテキストエディタ（メモ帳・VS Code など）で開く
2. 末尾の `}` の直前に新しい行を追加する
3. `"英語テキスト": "日本語テキスト",` の形式で記述する
4. ファイルを保存する
5. Chrome/Edge の拡張機能管理ページでリロードする（後述）

### 例：「Overview」を「概要」に翻訳したい場合

```json
{
  "Syncs": "同期",
  "Models": "モデル",
  "Overview": "概要"    ← これを追加
}
```

> ⚠️ **JSON の書き方のルール**
> - 各行の末尾にカンマ `,` が必要です（**最後の行だけカンマ不要**）
> - 文字列はダブルクォート `"` で囲む
> - 全角文字は使わない（コロン `:` は必ず半角）

---

## 翻訳を変更する

既存の翻訳が気に入らない場合は、右側の日本語テキストを書き換えます。

**変更前:**
```json
"Destination": "送信先",
```

**変更後:**
```json
"Destination": "配信先",
```

---

## 翻訳を削除する

該当の行ごと削除します。削除後、前の行の末尾カンマが不要にならないか確認してください。

---

## 変更を拡張機能に反映する方法

ファイルを編集・保存しただけでは拡張機能に反映されません。以下のいずれかの手順が必要です。

### 方法 A: 拡張機能をリロードする（推奨）

1. `chrome://extensions`（または `edge://extensions`）を開く
2. 「Hightouch 日本語化」の更新アイコン（🔄）をクリック
3. `app.hightouch.com` のタブをリロードする（Ctrl+R または ⌘+R）

### 方法 B: ページをリロードする

`app.hightouch.com` のタブをリロードするだけで反映される場合があります（キャッシュにより反映されない場合は方法Aを試してください）。

---

## 長い文字列を優先する理由

辞書に以下のような登録があるとします：

```json
"Sync now": "今すぐ同期",
"Sync": "同期"
```

もし短い `"Sync"` を先に処理すると、「Sync now」の「Sync」だけが「同期」に変換され「同期 now」になってしまいます。

このため、この拡張機能は**長い文字列のキーを先に処理する**設計になっています。`ja.json` に追加する順番を気にする必要はありません。

## カスタマイズを他のユーザーと共有する

良い翻訳を追加できた場合は、ぜひ Pull Request でメインリポジトリに貢献してください。詳細は [06. 貢献・共有方法](./06_contributing.md) を参照してください。

## 自社専用の翻訳辞書を持つ

リポジトリをフォークすることで、自社専用の翻訳辞書を独立して管理できます。例えば「Destination」を業界慣習に合わせて「媒体」と訳したい場合など、フォーク版でカスタマイズすることで対応できます。

フォーク方法は [06. 貢献・共有方法](./06_contributing.md) を参照してください。
