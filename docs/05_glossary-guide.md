# 05. 用語カタログの使い方

`translations/glossary.md` は、Hightouch UIの日本語翻訳における用語を統一するための公式リストです。このドキュメントでは、カタログの目的・読み方・編集方法を説明します。

---

## なぜ用語カタログが必要か

翻訳において「統一感」は非常に重要です。例えば「Destination」という用語一つをとっても：

- 「送信先」
- 「配信先」
- 「宛先」
- 「連携先」

など複数の候補があります。社内やクライアント間でバラバラな訳語を使うと、資料・UI・口頭説明が一致しなくなり混乱を招きます。

用語カタログは「この用語は必ずこの日本語で統一する」という組織のルールブックです。

---

## カタログの構成

`translations/glossary.md` は以下のセクションで構成されています：

| セクション | 内容 |
|-----------|------|
| コア機能 | Sync・Model・Destination など主要機能の用語 |
| ナビゲーション・メニュー | 画面上部・サイドバーのメニュー項目 |
| ステータス | Running・Failed など状態を表す用語 |
| アクション | Create・Save・Delete など操作ボタン |
| フォーム・入力 | フォームのラベル・プレースホルダー |
| テーブル・リスト表示 | 一覧画面の列ヘッダーやメッセージ |
| メッセージ・通知 | 確認ダイアログ・完了通知など |

---

## カタログの読み方

各用語は以下の形式で記載されています：

| 英語 | 日本語 | 品詞 | 備考 |
|------|--------|------|------|
| Sync | 同期 | 名詞・動詞 | 「Syncs」も「同期」（複数形は日本語で区別不要） |
| Destination | 送信先 | 名詞 | 配信先でも可だが「送信先」で統一 |

- **英語**: HightouchのUIに表示される原文
- **日本語**: この拡張機能で使用する公式訳
- **品詞**: 名詞・動詞・形容詞など（文脈による使い分けの参考）
- **備考**: 訳語選定の理由・注意事項・例外

---

## 翻訳の統一ルール

### カタカナを維持するケース

業界で定着しているマーケティング・データ用語はカタカナ表記を維持します：

- **Audience → オーディエンス**（「読者層」「受信者」ではなく業界標準のカタカナ）
- **Journey → ジャーニー**（「旅」「行程」ではなくCX用語として定着）
- **Pipeline → パイプライン**
- **Schema → スキーマ**

### 翻訳しない固有名詞

製品名・サービス名は翻訳しません：

- データウェアハウス: Snowflake、BigQuery、Redshift、Databricks
- CRM/MA: Salesforce、HubSpot、Marketo
- その他: Hightouch 本体、Slack、Google Ads など

## 用語の追加・変更を提案する

カタログに不足している用語や、既存の訳語に改善案がある場合は以下の手順で提案してください。

### 方法 1: GitHub Issue で提案（簡単）

1. [GitHub Issues](https://github.com/justin-hieher/hightouch-ja/issues) を開く
2. 「New Issue」をクリック
3. タイトルに「用語提案: ○○ → △△」と記載
4. 本文に変更理由を書いて送信

### 方法 2: Pull Request で直接編集（推奨）

1. リポジトリをフォークする
2. `translations/glossary.md` を編集する
3. Pull Request を作成する
4. Pull Request を確認のうえ、問題なければマージ（対応時期・可否は保証しません）

詳細は [06. 貢献・共有方法](./06_contributing.md) を参照してください。

## カタログと ja.json の関係

```
glossary.md（用語の定義・根拠）
      ↓ 定義に基づいて
ja.json（実際に適用される翻訳辞書）
```

`glossary.md` が「なぜこの訳語か」の根拠を持つカタログで、`ja.json` がその定義を実装したものです。`ja.json` を直接編集する際も、`glossary.md` の定義に沿った翻訳にしてください。

---

## 用語カタログを社内で活用する

このカタログは Hightouch UI の翻訳だけでなく、以下にも活用できます：

- **社内資料・提案書**: 「Destination」を「送信先」と統一して記載
- **顧客向け説明**: UI上の表示と一致した言葉で説明できる
- **新メンバーへのオンボーディング**: Hightouch の用語体系を一覧で把握できる
